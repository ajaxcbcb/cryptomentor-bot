Tujuan

SATUKAN logika analisis futures yang dipakai command /futures & /futures_signals ke satu modul reusable (engine).

AutoSignal memanggil engine yang sama untuk setiap koin top CMC, lalu mengirim hanya sinyal dengan confidence ≥ threshold (default 75).

Jaga output/format pesan konsisten dengan command manual (/futures, /futures_signals).

Tetap hormati premium/banned, DM consent, anti-spam cooldown, dan interval 30 menit.

0) Inventaris kode (AI Agent lakukan)
Cari implementasi command:

/futures handler

/futures_signals handler

Identifikasi fungsi/kelas yang menghitung:

arah (LONG/SHORT)

indikator (RSI/EMA/MACD/ADX/dst)

confidence (0–100)

Catat dependensi (Binance Futures klines, timeframe default, normalisasi symbol → BTCUSDT, dsb).

Kalau logika masih “nempel” di handler, ekstrak ke modul baru.

# app/autosignal.py
from app.futures.engine import analyze_symbol

def compute_signal_confidence(symbol: str) -> Optional[Dict[str, Any]]:
    """
    Panggil engine yang sama dengan /futures_signals.
    Return satu sinyal terbaik (confidence tertinggi) atau None.
    """
    try:
        res = analyze_symbol(symbol, timeframe=os.getenv("FUTURES_TF", "15m"), market=TARGET_MARKET)
        sigs = res.get("signals", []) or []
        if not sigs:
            return None
        best = max(sigs, key=lambda s: int(s.get("confidence", 0)))
        out = {
            "symbol": res.get("symbol") or f"{symbol}{TARGET_MARKET}",
            "side": best.get("side"),
            "confidence": int(best.get("confidence", 0)),
            "reason": ", ".join(best.get("reasons", []))[:300],
            "price": res.get("price"),
            "meta": res.get("meta", {}),
        }
        return out
    except Exception:
        return None
Threshold kirim tetap dipakai dari MIN_CONFIDENCE (default 75).

5) Rate limit & error handling
Pastikan engine punya caching klines pendek (mis. 15–30 dtk in-memory) agar 10–15 koin tidak mem-banjiri API.

Tambahkan timeout dan retry ringan ketika ambil data exchange.

Kalau ada error satu koin, skip saja; jangan hentikan keseluruhan batch.

6) Eligibility & pengiriman (tetap)
Hanya kirim ke user premium aktif/lifetime, tidak banned, punya chat privat (consent).

Anti-spam: cooldown per (symbol, side) minimal COOLDOWN_MIN menit.

Interval scheduler 30 menit (sudah kamu minta).

7) Admin controls & debug
/signal_tick → jalankan scan sekali, balas jumlah sinyal terkirim.

/signal_status → tampilkan ON/OFF, interval, top_n, threshold, TF.

/signal_on / /signal_off → toggle fitur.

(Opsional) /signal_trace <symbol> → tampilkan debug dari analyze_symbol (tanpa DM) untuk satu coin (admin only).

8) Acceptance test
Jalankan /futures BTC 15m dan /futures_signals BTC 15m → catat side/confidence.

Jalankan /signal_tick:

Untuk top coins (BTC/ETH), AutoSignal harus menghasilkan sinyal yang konsisten (side/confidence sama) dengan command manual, karena keduanya now calling the same engine.

Turunkan sementara threshold AUTOSIGNAL_MIN_CONF=60 untuk memastikan pesan terkirim saat uji, lalu kembalikan ke 75.

Cek user premium yang sudah /start privat menerima DM sinyal.
